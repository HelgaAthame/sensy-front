# План улучшений фронта (sensy_front)

Живой список задач по фронту. Пункты добавляются по мере поступления (в основном — от тебя), реализуются по одному и отмечаются как сделанные прямо здесь же.

## Как пользоваться

- Новые пожелания — в раздел **«Бэклог»** ниже, коротко, своими словами, я разберусь с деталями.
- Когда пункт реализован — переезжает в **«Сделано»** с кратким описанием, что именно поменялось.

## Сделано

- [x] Валидация пароля на форме **входа** была скопирована с формы регистрации (проверяла сложность нового пароля — заглавная буква, цифра — вместо простого «введите текущий пароль»), из-за чего валидный пароль с бэка не проходил. Убрана лишняя проверка, осталось только «поле не пустое».
- [x] `createMediaFile` ожидал в ответе массив (`MediaFileResponse[]`) на загрузку одного файла — исправлено под контракт бэка, который теперь возвращает один объект `MediaFile`.
- [x] Next.js обновлён `15.2.4 → 15.2.8` — закрыта критическая RCE-уязвимость в React Server Components (GHSA-9qr9-h5gf-34mp / CVE-2025-55182 / CVE-2025-66478), официальный security-фикс от Vercel.

## Бэклог

- [ ] **Адаптация под узкие экраны.** Сейчас `AuthGate.tsx` жёстко блокирует всё приложение ниже 1024px, показывая только текст «Это приложение доступно только на экранах шириной от 1024px» — заглушка, ничего не отображает. При этом разметка уже частично готова к мобильному режиму: у `Sidebar`/`Header`/`SidebarContext` есть механизм off-canvas меню (`isMobileOpen`, бургер-кнопка, backdrop), просто он не используется, потому что `useMinWidth(1024)` в `AuthGate` не даёт до него дойти. Похоже, основная работа — снять/понизить блокировку и точечно починить то, что реально ломается на узких экранах (в первую очередь широкие таблицы — звонки/операторы/проекты/чек-листы/словари, возможно графики аналитики), а не строить адаптив с нуля.
- [ ] **Ошибка загрузки звонка «Input not instance of File»** — нечитаемый текст ошибки на форме загрузки записи. Нужно: (а) заменить на человекопонятное сообщение, (б) показывать его рядом с полем выбора файла, а не над кнопкой отправки.
- [ ] **Dropzone показывает подсказку поверх уже загруженного файла.** Текст «Поместите файл сюда / Перетащите вашу аудиозапись в формате WAV, MP3 сюда или выберите на вашем устройстве» должен скрываться, когда файл уже выбран/загружен — сейчас видны одновременно и подсказка, и загруженный файл.
- [ ] **Профиль пользователя и настройки аккаунта.** Сейчас на фронте (и в контракте бэка) нет ни страницы профиля, ни настроек — нужно продумать состав (как минимум смена пароля, отображение email/роли) и куда это добавить в навигации; потребует новых эндпоинтов на бэке.
- [ ] **Единообразные красивые лоадеры.** Сейчас индикатор загрузки есть только при загрузке списков — нужно привести все состояния загрузки по всему приложению к одному согласованному, визуально аккуратному компоненту-лоадеру, а не оставлять часть мест вообще без индикации.
- [ ] **Экран "пробуждения" бэка.** Бэк на Render free tier засыпает после простоя, первый запрос после сна занимает 30-60с (холодный старт). Решено: не держать бэк вечно активным внешним пингером (съело бы почти весь бесплатный лимит часов Render), а на фронте при открытии сайта сразу пинговать `GET /health` и показывать дружелюбный экран ожидания ("сервер просыпается...") до первого успешного ответа, вместо того чтобы просто зависать/показывать ошибки таймаута.
